# Screenshots

Inventory of documentation screenshots. Images live in `docs/assets/screenshots/`. Screenshots come from three sources: the repeatable capture script (`npm run screenshots:docs`, which runs `scripts/capture-docs-screenshots.js` against a disposable WordPress Playground — no Docker needed), images that already existed in the repository (`img/*.png`, matching the readme.txt screenshot captions), and rows marked "manual needed". For the remaining manual rows, the repository's Playwright visual-regression suite (`tests/e2e/visual-regression.spec.js` with `tests/js/sample-values.js`) and the WordPress Playground blueprint (`blueprints/blueprint.json`, with a screenshot recipe in `blueprints/README.md`) can drive most captures with populated demo data.
