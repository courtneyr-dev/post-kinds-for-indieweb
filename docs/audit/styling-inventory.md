Generated by audit Session C1, 2026-05-01. Read-only inventory.

styling value in the plugin source so that Session C2 can replace them with
CSS custom property tokens that default to neutral.

`patterns/` — 18 CSS files, 70 PHP files, 71 JS files, 38 `block.json` files.

contains every violation with file, line, surface, kind, and value. Use
`jq` for ad-hoc queries during C3 migration.

## Star rating — Section 5.4 special case

The audit prompt called out `#ffc107` (gold) as the canonical example.
Confirmed present at `src/blocks/star-rating/style.css:8`.

**Full violation list for star-rating:**

| File | Line | Kind | Value | Note |
|---|---|---|---|---|
| `src/blocks/star-rating/editor.css` | 19 | font-family | `-apple-system, BlinkMacSystemFont, "Segoe UI Symbol", system…` | System font stack for star glyphs — keep for cross-platform symbol rendering |

| `src/blocks/star-rating/style.css` | 8 | hex | **`#ffc107`** | **The gold star fill — tokenize as `--pkiw-star-rating-fill, currentColor`** |
| `src/blocks/star-rating/style.css` | 9 | hex | **`#ddd`** | **Empty star fill — tokenize as `--pkiw-star-rating-empty, transparent`** |
| `src/blocks/star-rating/style.css` | 34 | font-family | `-apple-system, BlinkMacSystemFont, "Segoe UI Symbol", system…` | Same as editor — keep |
| `src/blocks/star-rating/style.css` | 74 | hex | `#e91e63` | Pink variant (color modifier rating?) — tokenize per modifier |
| `src/blocks/star-rating/style.css` | 78 | hex | `#2196f3` | Blue variant — tokenize per modifier |
| `src/blocks/star-rating/style.css` | 84 | hex | `#666` | Caption text — tokenize as `--pkiw-star-rating-caption, inherit` |

**C2 token design for star-rating:**

```css
:root {
  --pkiw-star-rating-fill: currentColor;     /* Was #ffc107 */
  --pkiw-star-rating-empty: transparent;     /* Was #ddd */
  --pkiw-star-rating-caption: inherit;       /* Was #666 */
  --pkiw-star-rating-border-radius: 0;       /* Was 2px on focus indicator */
  --pkiw-star-rating-variant-1: currentColor; /* Was #e91e63 */
  --pkiw-star-rating-variant-2: currentColor; /* Was #2196f3 */
}
```

System font stack (`-apple-system, BlinkMacSystemFont, "Segoe UI Symbol"`)
stays as-is — the audit prompt's contract is about color/font choice; star
glyphs need a font that has the right Unicode coverage, which is a
functional choice, not a design choice. Document the exemption.

## C2 token design preview (for reference)

Per Section 5.2, the `--pkiw-*` namespace. Categories implied by the
inventory:

```css
/* Card surface */
--pkiw-card-bg
--pkiw-card-fg
--pkiw-card-border
--pkiw-card-shadow
--pkiw-card-border-radius

/* Per-kind paint tokens (Section 5.2 prompt list — 17 cards × 3-4 tokens each) */
--pkiw-{kind}-card-bg, --pkiw-{kind}-card-fg, --pkiw-{kind}-card-accent
--pkiw-{kind}-artwork-ratio

/* Star rating */
--pkiw-star-rating-fill, --pkiw-star-rating-empty, --pkiw-star-rating-caption

/* Stats blocks (the three greens) */
--pkiw-stat-positive, --pkiw-stat-neutral, --pkiw-stat-active

/* Layout (structural — small subset of border-radius/spacing tokens) */
--pkiw-card-gap, --pkiw-card-padding

/* Editor chrome — exempted from the contract but still tokenized */
--pkiw-focus-indicator (was #007cba)
```

Per Section 5.2's prompt: every paint token defaults to neutral
(`transparent` / `inherit` / `currentColor`) so themes that don't opt in
get cards with no painted background.

## Open questions for C2

1. **The three greens** (`#127925`, `#127916`, `#128214`) in `class-media-stats.php` and `class-now-playing.php` — what do they semantically represent? The values are too close visually to be different states; either rename to a single `--pkiw-stat-color` OR confirm three distinct semantic roles for three distinct tokens.

2. **`#007cba`** appears in `patterns/read-progress.php:96-97` (3 times) and `src/blocks/star-rating/editor.css:29`. WP admin blue. In `read-progress.php` it's user-facing (rendered HTML) — should be tokenized. In `star-rating/editor.css` it's the focus indicator on the editor surface — could stay as admin chrome OR tokenize as `--pkiw-focus-indicator`.

3. **System font stack** for star-rating glyphs (`-apple-system, BlinkMacSystemFont, "Segoe UI Symbol"`). Functional choice (Unicode coverage), not design choice. Recommend exempting from the contract; document the exemption in C2's `DESIGN-TOKENS.md`.

4. **Admin-page inline styles (115 violations).** Audit prompt's contract is silent on admin UI. Recommend admin pages keep their WP-admin-color-scheme integration (`#2271b1`, `#46b450`, etc. — these are documented WP admin colors). C2 should explicitly carve out the admin surface in `THEME-INTEGRATION.md`.

5. **`src/blocks/shared/card-editor.css` (126 violations) is the largest target.** C3 may want to do this migration as its own session (C3a) before tackling per-block stylesheets, since fixing the shared stylesheet may eliminate violations in downstream files.
